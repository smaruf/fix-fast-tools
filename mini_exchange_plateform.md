For your use case, you are effectively looking for a **mini exchange platform**:

```text
                 +------------------+
FIX Client ----> | FIX Gateway      |
                 +------------------+
                          |
                          v
                 +------------------+
                 | Matching Engine  |
                 +------------------+
                    |          |
                    v          v
             Execution     Market Data
               Reports      Publishers
                             |      |
                             v      v
                           ITCH    FAST
```

Unfortunately, there is no widely used open-source Java JAR that provides all three together:

* FIX Order Entry
* Matching Engine
* ITCH Broadcaster
* FAST Broadcaster

out of the box.

### What I would recommend

#### FIX Gateway

Use [QuickFIX/J](https://www.quickfixj.org/?utm_source=chatgpt.com) as the exchange-side acceptor. It supports FIXT 1.1 and FIX 4.x/5.x and even ships with an Executor example that simulates order execution. ([QuickFIX/J][1])

#### Matching Engine

Use [Exchange Core](https://github.com/exchange-core/exchange-core?utm_source=chatgpt.com) for realistic order-book matching. It provides price-time-priority matching and high-throughput order processing. ([GitHub][2])

#### FAST Publisher

Use Java OpenFAST:

[OpenFAST](https://openfast.sourceforge.net/?utm_source=chatgpt.com)

OpenFAST is a Java implementation of the FAST protocol and can encode market data generated by your matching engine. FAST is specifically designed for compressed high-volume market data distribution. ([Wikipedia][3])

#### ITCH Publisher

There is no mature Java Nasdaq ITCH simulator that is commonly used. Most teams implement:

```java
OrderBookEvent
      ↓
ITCH Encoder
      ↓
UDP Multicast
```

The ITCH message set is relatively small compared with FIX, so implementing:

* System Event
* Stock Directory
* Add Order
* Order Executed
* Order Cancel
* Order Replace
* Trade

is usually only a few hundred lines of encoder code.

### Fastest Path (1–2 days)

Since you already implemented DSE/Nasdaq FIX:

```text
QuickFIX/J Acceptor
        |
        v
Simple Matching Engine
        |
        +--> ExecutionReport (35=8)
        |
        +--> ITCH UDP Multicast
        |
        +--> FAST UDP Multicast
```

This gives you:

* Real FIX sessions
* Order acknowledgements
* Rejects
* Partial fills
* Full fills
* ITCH market data feed
* FAST market data feed

for local testing.

### Commercial-Grade Simulator

If you want something ready-made instead of coding:

* [PhiFIX Simulator](https://www.phifix.com/simulator/?utm_source=chatgpt.com) supports FIX, FAST, ITCH/OUCH, exchange simulation, order matching, and market-data generation. ([Phifix][4])

However, it is commercial software.

Kafka
    +

and package it as a single Spring Boot application. That would behave much closer to a real Nasdaq-style exchange than most demo simulators and would let your existing FIX client connect without modification.
